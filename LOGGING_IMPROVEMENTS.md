# Улучшения логирования для диагностики видео обработки

## Дата: 17 декабря 2025

### Проблема
При обработке видео файлов (особенно webm) система молчаливо не получала аудио из видео. Ошибки ffmpeg были подавлены флагом `SUPPRESS_FAILURE_MESSAGES`, что затрудняло диагностику.

### Решение
Добавлено детальное логирование на нескольких уровнях для отслеживания процесса обработки видео.

---

## 1. Улучшения в `transkribator_modules/audio/extractor.py`

### Что добавлено:
- **Проверка файла перед обработкой**: логирование размера видео-файла
- **ffprobe pre-check**: проверка наличия аудио потока ДО попытки извлечения
  - Выполняется запрос `ffprobe` с параметрами `-select_streams a:0` для поиска audio stream
  - Логируется результат: наличие/отсутствие аудио потока
  - Логируется stderr ffprobe при проблемах
  
- **Полное логирование ffmpeg ошибок**:
  - `return_code` процесса ffmpeg
  - Первые 1000 символов stderr (полная информация об ошибке)
  - stdout для дополнительного контекста
  
- **Валидация результата**:
  - Проверка, что файл создан и имеет размер > 0
  - Логирование размера извлеченного аудио файла
  
- **Обработка исключений**:
  - Логируется тип исключения (Exception type)
  - Timeout ошибки обрабатываются отдельно

### Примеры логов:
```
INFO: Извлечение аудио
  video: "/app/videos/telegram_video_XXXXX.mp4"
  audio: "/app/audio/telegram_audio_XXXXX.wav"
  video_size_mb: 173.2

INFO: ffprobe результат
  probe_output: "audio"
  return_code: 0

INFO: Аудио успешно извлечено
  audio: "/app/audio/telegram_audio_XXXXX.wav"
  size_mb: 245.3
  original_video_mb: 173.2

WARNING: Видео не содержит аудио потока
  probe_stdout: ""
  probe_stderr: "[stderr content from ffprobe]"

ERROR: Ошибка при извлечении аудио ffmpeg
  return_code: 1
  stderr: "[ffmpeg error details - first 1000 chars]"
```

## 2. Улучшения в `transkribator_modules/bot/handlers.py`

### 2.1 Document handler логирование (строка ~2500)
- Добавлено логирование MIME type документа
- Добавлено логирование размера файла в МБ
- Помогает различать как документ был отправлен

Пример:
```
INFO: Получен видео-документ от пользователя 376134276
  file_size_mb: 173.2
  mime_type: "video/webm"
```

### 2.2 process_video_file ошибки извлечения (строка ~530)
- Добавлены более информативные сообщения об ошибке пользователю
- Улучшено логирование с дополнительным контекстом:
  - `file_name`: имя файла
  - `file_size_mb`: размер для понимания проблем с памятью
  - Ошибка все еще SUPPRESS_FAILURE_MESSAGES, но информация есть в логах

Пример сообщения:
```
❌ Не удалось извлечь аудио из видео. Возможно, видео без звука или в неподдерживаемом формате.
```

### 2.3 Очистка файлов логирование (строка ~783)
- Добавлено логирование факта удаления каждого файла
- Помогает отследить, на каком этапе был очищен файл
- Логируется тип ошибки при неудаче очистки

Примеры:
```
INFO: Удален видео-файл: /app/videos/telegram_video_XXXXX.mp4
INFO: Удален аудио-файл: /app/audio/telegram_audio_XXXXX.wav
INFO: Удален сжатый аудио-файл: /app/audio/compressed_XXXXX.wav
```

## 4. Диагностика

### Как отследить проблему с видео:

1. **Пользователь отправляет видео**
   ```
   LOG: "Получен видео-документ от пользователя 376134276: video.webm"
   ```

2. **Скачивание файла**
   ```
   LOG: "Извлечение аудио"
        video: "/app/videos/telegram_video_XXXXX.mp4"
        video_size_mb: 173.2
   ```

3. **Проверка аудио потока**
   ```
   LOG: "ffprobe результат"
        probe_output: "" (пусто если нет аудио)
        return_code: 0
   
   WARNING: "Видео не содержит аудио потока" (если пусто)
   ```

4. **Попытка извлечения**
   ```
   ERROR: "Ошибка при извлечении аудио ffmpeg"
          stderr: "[детали ошибки ffmpeg]"
   ```

5. **Очистка файлов**
   ```
   INFO: "Удален видео-файл: /app/videos/..."
   ```

### Поиск ошибок в логах:
```bash
# Все ошибки извлечения аудио за последний час
docker logs --since 1h cyberkitty19-transkribator-bot 2>&1 | grep -A5 "Audio extraction failed"

# Все ffmpeg ошибки
docker logs cyberkitty19-transkribator-bot 2>&1 | grep -E "ffmpeg|ffprobe"

# Логи для конкретного пользователя
docker logs cyberkitty19-transkribator-bot 2>&1 | grep "376134276"
```
