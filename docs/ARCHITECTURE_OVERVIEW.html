CyberKitty19 · Архитектура и взаимодействия

Подробное описание контейнеров, пайплайнов, очередей, мини‑приложения и пользовательских сценариев. Диаграммы на базе Mermaid показывают связи компонентов и пошаговую обработку медиаконтента/поисковых запросов.

1 · Контейнеры и сервисы

bot

cyberkitty19-transkribator-bot

  • PTB, polling через локальный Bot API.
  • Скачивание медиа, конвертация, локальный Whisper/TranscribeClient.
  • Создание заметок, отправка summary/файлов.
  • enqueue MediaJob в processing_jobs.
worker

cyberkitty19-transkribator-worker

  • Pipelines run_media_pipeline.
  • auto_finalize_note + IndexService (pgvector).
  • Whisper сегменты, fallback hash embeddings.
api

cyberkitty19-transkribator-api

  • FastAPI/uvicorn, MiniApp REST, поиск, оплаты.
  • Реиспользует transkribator_modules (NoteService, ReferralService).
db

PostgreSQL

  • Бизнес-данные: users, notes, transcriptions, plans, transactions.
  • Технические: processing_jobs, note_chunks, note_groups…
whisper

real-whisper

  • Uvicorn сервис, локальная модель Whisper.
  • Отдаёт сегменты, используется ботом и worker’ом.
aux

Telegram Bot API Server

  • Локальный instance для надёжного скачивания медиа.
  • Снимает ограничения и уменьшает latency.

4 · Подсистемы и конфигурации

Подсистема Роль Ключевые настройки
Transcription Whisper (локально) + fallback OpenAI; кеширование сегментов. TRANSCRIBE_DEFAULT_MODE, TRANSCRIBE_CLIENT_ENABLED.
Summary generate_brief_summary через OpenRouter/Gemini. OPENROUTER_API_KEY, ROUTER_MODEL=gemini-2.5.
Queue PostgreSQL processing_jobs, worker с backoff. JOB_LOCK_TIMEOUT_SECONDS, APScheduler для sync задач.
Embeddings OpenRouter/DeepInfra, кеш, fallback hash. EMBEDDING_PROVIDER, EMBEDDING_MODEL, DISABLE_REMOTE_EMBEDDINGS.
Vector Search pgvector cosine + full-text hybrid + rerank. ENABLE_HYBRID_SEARCH, HYBRID_VECTOR_WEIGHT, ENABLE_RERANKING.
MiniApp WebApp UI, просмотр/правка заметок, поиск. MINIAPP_PUBLIC_URL, MINIAPP_NOTE_LINK_TEMPLATE.
Payments & Referrals Тарифы, подписки, промокоды, рефералка. ENV для CloudPayments, Stripe, YooKassa, Telegram Stars.

5 · Пользовательские флоу

5.1 Загрузка аудио/видео

  1. Upload: пользователь отправляет медиа в чат; бот отвечает “Файл принят! Готовлю обработку…”.
  2. Progress: бот рисует прогресс‑бар (10 сегментов) пока скачивает/конвертирует.
  3. Transcribe: Whisper/TranscribeClient → текст, таймкоды, summary.
  4. Result Screen: “✅ Обработка завершена!” + summary + кнопки:
    • 📄 Скачать текст (TXT)
    • 🔎 Задать вопросы (режим vector chat)
    • 🏠 Главное меню
  5. Sync: job в очередь; worker доиндексирует заметку.

5.2 Задать вопрос по файлу

  1. Пользователь нажимает “🔎 Задать вопросы”.
  2. Bot → state qa_session; последующие сообщения идут в vector chat.
  3. Система строит запрос в IndexService (учитывая дату, теги, группы) → top‑chunks.
  4. Формируется системный промпт (описание данных + инструкции) + вопрос; отправляется в выбранную LLM (OpenRouter/Gemini).
  5. Ответ (“что обсуждали”, “какие ошибки”) отправляется пользователю; ссылки на источники удалены.

5.3 MiniApp / просмотр заметок

  1. Кнопка “📒 Личный кабинет” открывает WebApp.
  2. MiniApp авторизуется через Telegram initData, вызывает API container.
  3. Пользователь просматривает список заметок, фильтры по тегам/датам, запускает поиск, редактирует summary.
  4. Доступны тарифы/подписки, рефералка, настройки “формат расшифровки / модель / язык”.

5.4 Интеграция с Google Drive

  1. APSheduler job beta_drive_sync каждые 10 минут проверяет папку Drive “CyberKitty”.
  2. Создаёт отсутствующие папки, выгружает новые заметки/файлы.
  3. В логах фиксируются ошибки (например, таймаут прокси YTDLP_PROXY).

7 · Логи и мониторинг

  • Bot: DEBUG‑логи HTTPX, события Creating note..., Enqueued media job..., ошибки скачивания.
  • Worker: Processing job, прогресс стадий, предупреждения embedding fallback, Drive sync.
  • API: журналы FastAPI/uvicorn, ошибки MiniApp, платежи.
  • DB: отслеживание состояния очереди, note_chunks, метрик через pg_stat.
